◆ push()

template<typename T >
bool ThreadsafeQueue< T >::push ( T  new_value)
overridevirtual

Push by value. Returns false if the queue has been shutdown. Not optimal, since it will make two move operations. But it does the job: see Item 41 Effective Modern C++.

Alternatives: Using both:

  • push(const T&)
  • with push(T&& ) rvalue to the queue using move semantics. Since there is no type deduction, T&& is NOT a universal reference (typename T is not at the level of the push function). Problem: virtual keyword will make push(T&&) be discarded for push(const T&), non-copyable things will compile-complain.
