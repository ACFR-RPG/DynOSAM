◆ has_motion_pair

template<typename MAP >
bool dyno::ObjectUpdateContext< MAP >::has_motion_pair {false}

Indicates that we have a valid motion pair from k-1 to k (this frame) and therefore k is at least the second frame for which this object has been consequatively tracked When this is false, it means that the frame k-1 (getFrameId() - 1u) did not track/observe this object and therfore no motion can be created between them. This happens on the first observation of this object.
